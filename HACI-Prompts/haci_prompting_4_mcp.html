The Three MCP Primitives

Model Context Protocol exposes three primitives from servers to clients. Prompts are often overlooked but provide powerful instruction templating capabilities.

üîß
Tools

Executable functions that agents can invoke. The most commonly used MCP primitive.

  • Execute queries
  • Modify resources
  • Trigger actions
üìÅ
Resources

Structured data sources accessible via URI patterns. Provide context to agents.

  • Configuration data
  • Documentation
  • System state
üìù
Prompts

Instruction templates with arguments. Enable servers to guide agent behavior dynamically.

  • Analysis templates
  • Investigation patterns
  • Best practice guidance
üí° Why MCP Prompts Matter for HACI
MCP Prompts allow external systems to contribute specialized instructions to HACI agents. Instead of hardcoding all investigation patterns, servers like GitHub, Jira, or Datadog can expose domain-specific prompts that agents retrieve dynamically. This enables vendor-specific best practices without modifying HACI's core code.

Static vs Dynamic Prompts

Understanding when to use hardcoded HACI prompts versus MCP-provided prompts.

Prompt Strategy Comparison
Aspect Static (HACI System Prompts) Dynamic (MCP Prompts)
Location Hardcoded in agent configuration Retrieved from MCP servers at runtime
Update Cycle Deployment Required Instant Updates
Customization Per-organization customization Per-vendor/tool specialization
Best For Core agent identity, harness instructions Vendor-specific patterns, external best practices
Token Cost Always loaded Loaded on-demand
Reliability Always available Depends on server availability
üîÑ HACI's Hybrid Approach
HACI uses static prompts for core agent identity, harness instructions, and governance controls. MCP prompts supplement with vendor-specific investigation patterns. If an MCP server is unavailable, agents fall back to generic investigation patterns.

Implementing MCP Prompts in HACI

How HACI agents discover, retrieve, and apply MCP prompts during investigations. 

# haci/agents/mcp_prompt_integration.py

class MCPPromptIntegration:
    """
    Integrates MCP prompts into agent investigations
    """
    
    async def get_investigation_prompt(
        self,
        server_name: str,
        prompt_name: str,
        arguments: Dict[str, Any]
    ) -> Optional[str]:
        """
        Retrieve an investigation prompt from an MCP server
        
        Args:
            server_name: Name of the MCP server (e.g., "github")
            prompt_name: Name of the prompt template
            arguments: Arguments to fill the template
            
        Returns:
            Filled prompt template or None if unavailable
        """
        try:
            # Get MCP client for this server
            client = self.mcp_manager.get_client(server_name)
            
            # Retrieve prompt with arguments
            response = await client.prompts.get(
                name=prompt_name,
                arguments=arguments
            )
            
            # Extract the prompt content
            if response.messages:
                return response.messages[0].content
            
        except MCPServerUnavailable:
            # Fall back to generic pattern
            logger.warning(f"MCP server {server_name} unavailable, using fallback")
            return self._get_fallback_prompt(prompt_name)
        
        return None
    
    async def enrich_think_phase(
        self,
        base_prompt: str,
        ticket_context: Dict
    ) -> str:
        """
        Enrich THINK phase with relevant MCP prompts
        """
        enrichments = []
        
        # If ticket mentions GitHub PR, get PR analysis prompt
        if "pr_number" in ticket_context:
            pr_prompt = await self.get_investigation_prompt(
                server_name="github",
                prompt_name="analyze_pr",
                arguments={"pr_number": ticket_context["pr_number"]}
            )
            if pr_prompt:
                enrichments.append(pr_prompt)
        
        # If ticket has Datadog alert, get metric investigation prompt
        if "alert_id" in ticket_context:
            metric_prompt = await self.get_investigation_prompt(
                server_name="datadog",
                prompt_name="metric_investigation",
                arguments={"alert_id": ticket_context["alert_id"]}
            )
            if metric_prompt:
                enrichments.append(metric_prompt)
        
        # Combine base prompt with enrichments
        return base_prompt + "\n\n## Vendor-Specific Guidance\n" + "\n\n".join(enrichments)

MCP Prompt Best Practices

‚ö†Ô∏è Security Considerations
MCP prompts come from external servers. HACI validates all retrieved prompts:
  • Size limits: Reject prompts exceeding 10K tokens
  • Content scanning: Check for injection attempts
  • Allowlisting: Only accept prompts from registered servers
  • Fallback behavior: Always have a safe fallback if prompts fail validation
‚úÖ Design Guidelines for Custom MCP Prompts
When creating custom MCP servers for HACI:
  • Keep prompts focused: One investigation pattern per prompt
  • Include context hints: Tell agents what data they're looking at
  • Provide output guidance: Suggest output formats compatible with HACI
  • Version your prompts: Include version metadata for debugging
  • Document arguments clearly: Make required vs optional obvious
